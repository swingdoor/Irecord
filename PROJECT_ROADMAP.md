# 本地语音识别转写系统 - 项目路线图

**项目名称**: 语音转写助手  
**技术栈**: Electron + React + Sherpa-ONNX  
**目标平台**: Windows 10/11  
**开发周期**: 13-15 周

---

## 📋 项目概述

开发一款基于 Windows 系统的本地语音识别转写软件，支持音频/视频文件批量处理，提供高精度的语音转文字、说话人分离、时间戳对齐、关键词提取等功能。

**核心特性**:
- ✅ 完全本地处理，无需联网
- ✅ 支持 CPU 和 GPU 两种模式
- ✅ 多模型支持（Qwen3-ASR-0.6B、SenseVoice、Paraformer）
- ✅ 说话人分离（2-10 人）
- ✅ 词级时间戳
- ✅ 自动标点预测
- ✅ 关键词提取
- ✅ 多格式导出（TXT/SRT/VTT/JSON/Word/PDF）

## 🎯 Change 拆分策略

### Change 1: MVP - 基础语音识别 ⭐⭐⭐⭐⭐
**优先级**: P0  
**工作量**: 2-3 周  
**依赖**: 无

**功能范围**:
- Electron 应用框架（React + TailwindCSS）
- 文件上传（拖放 + 选择，支持 MP3/WAV/MP4）
- Sherpa-ONNX 集成（仅 CPU 模式）
- Qwen3-ASR-0.6B 模型（单一模型）
- 基础识别（文本 + 词级时间戳）
- 简单进度显示
- 结果展示（纯文本）
- TXT 导出
- 基础错误处理

**交付标准**:
- 能识别 10 分钟以内的音频文件
- 中文识别基本准确（CER < 8%）
- 能导出 TXT 文件
- 无明显 bug

---

### Change 2: 说话人分离 ⭐⭐⭐⭐
**优先级**: P0  
**工作量**: 2 周  
**依赖**: Change 1

**功能范围**:
- 集成 pyannote-segmentation-3.0
- 集成 3D-Speaker embedding
- 说话人聚类算法
- ASR 结果与说话人对齐
- 说话人标记 UI（颜色区分）
- 说话人统计信息
- 带说话人的 TXT 导出

**交付标准**:
- 2-5 人场景说话人分离准确率 > 85%
- 说话人 ID 稳定（不频繁切换）
- UI 清晰展示不同说话人

---

### Change 3: 标点预测 + 关键词提取 ⭐⭐⭐
**优先级**: P1  
**工作量**: 1-2 周  
**依赖**: Change 1

**功能范围**:
- CT-Transformer 标点预测模型
- 自动添加标点符号（逗号、句号、问号、感叹号）
- nodejieba 中文分词

- 关键词 UI 展示（标签云/列表）
- 关键词搜索和高亮
- 自定义关键词词典支持

**交付标准**:
- 标点预测准确率 > 90%
- 关键词提取合理（前 10 个关键词相关性高）
- 支持点击关键词跳转到对应位置

## 🏗️ 技术架构

### 核心技术栈
- **前端**: Electron 32 + React 18 + TailwindCSS 3
- **后端**: Node.js + Sherpa-ONNX 1.12+
- **引擎**: ONNX Runtime (CPU/GPU)
- **构建**: Vite 5 + electron-builder 25

### 模型清单
| 模型 | 大小 | 用途 | 优先级 |
|------|------|------|--------|
| Qwen3-ASR-0.6B | ~600MB | ASR（主推荐） | P0 |
| SenseVoice-Small | ~200MB | ASR（备选） | P1 |
| Paraformer-large-zh | ~300MB | ASR（备选） | P1 |
| pyannote-segmentation-3.0 | ~17MB | 说话人分割 | P0 |
| 3D-Speaker | ~5MB | 说话人嵌入 | P0 |
| CT-Transformer | ~40MB | 标点预测 | P1 |
| Silero VAD | ~2MB | 语音活动检测 | P0 |

**总计（默认配置）**: ~670MB
