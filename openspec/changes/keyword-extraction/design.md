## Context

当前应用已有完整的识别链路：音频上传 → 可选 VAD/说话人分离 → ASR 识别 → 结果页展示与导出。结果页当前只显示完整文本和说话人信息，用户仍需手动阅读全文才能抓取重点。

关键词提取属于纯文本后处理功能，不依赖音频模型，可在主进程或子进程完成。考虑到转写结果已经在主进程拿到，关键词提取放在主进程最简单。

## Goals / Non-Goals

**Goals:**
- 从转写文本中提取关键词
- 支持中文文本
- 关键词按重要性排序
- 结果页展示关键词，支持点击定位/高亮
- 导出时可附带关键词摘要

**Non-Goals:**
- LLM 摘要生成
- 多语言关键词提取优化
- 自动分类/主题建模

## Decisions

### 1. 使用 TF-IDF + 中文分词

**决策**：使用 `nodejieba` 做中文分词，基于词频和简单停用词过滤提取关键词。

**理由**：
- 轻量、稳定、本地运行
- 无需额外模型
- 对当前中文转写场景足够

**备选方案**：
- ❌ KeyBERT / embedding 模型：太重，不适合当前桌面本地场景
- ❌ 纯词频统计：没有中文分词效果差

### 2. 关键词提取位置：主进程后处理

**决策**：ASR 结果返回主进程后，在 `ipc.ts` 中调用关键词提取函数。

**理由**：
- 关键词提取是纯文本操作，不需要子进程
- 逻辑简单，易于调试

### 3. 结果页交互

**决策**：关键词显示为标签列表，点击后高亮结果页中的匹配文本片段。

## Risks / Trade-offs

### 1. 中文分词误差
**风险**：转写错误会影响分词和关键词质量。
**缓解**：优先过滤单字、停用词和过短词。

### 2. 关键词过多/过少
**风险**：不同文本长度下关键词数量不稳定。
**缓解**：默认输出 Top 10，可后续配置。
